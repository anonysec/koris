# Address code review issues (v2)

This commit fixes four of six issues flagged in the v1 review: architecture detection now uses `uname -m`, Go binary downloads include SHA256 checksum verification, `resetToDefaults` has a guard against missing `PROTOCOL_DEFAULTS` keys, and `formatTime` uses `Intl.RelativeTimeFormat` for locale-aware relative timestamps. The two remaining v1 issues (VPN disconnect dead code, notification store mutation) were confirmed to be pre-existing code outside these 5 commits and are out of scope.

**Watch for:** The Go checksum verification queries `go.dev/dl/?mode=json` which only returns the two latest stable releases — since the pinned version is Go 1.22.5 (already superseded by 1.23.x), the checksum lookup will always return empty, making verification effectively dead code that falls through to the "Proceeding with install" warning path (confirmed). The `resetToDefaults` error toast uses a hardcoded English string rather than a `t()` key, inconsistent with the i18n pass applied everywhere else in this commit (confirmed).

**Verdict**: APPROVED

## High-level view

The architecture detection fix uses a `uname -m` case mapping covering x86_64, aarch64/arm64, and armv7l with a safe fallback to amd64.

The checksum verification structure is correct (download to file, hash, compare, abort on mismatch) but the hash source is unreliable for the pinned Go version. The `?mode=json` endpoint only serves current releases, and Go 1.22.5 is no longer current. The fallback path logs a warning and proceeds without verification — in practice verification will never succeed until the pinned version is bumped or the URL changed to `?mode=json&include=all`.

The locale-aware `formatTime` uses `Intl.RelativeTimeFormat` with the reactive `locale` ref. The `numeric: 'auto'` option means `rtf.format(0, 'second')` produces "now" in English and equivalents in FA/ZH rather than "in 0 seconds".

The `resetToDefaults` guard's error message is English-only (`'No defaults available for this protocol'`), inconsistent with the i18n pass in the same commit.

## Go checksum verification will silently skip on pinned version

The checksum retrieval fetches from `https://go.dev/dl/?mode=json` and greps for the filename pattern `go1.22.5.linux-amd64`. The Go download API's `?mode=json` endpoint only returns metadata for the two most recent stable releases (currently in the 1.23.x line). Since Go 1.22.5 is no longer listed, the grep returns empty, `EXPECTED_HASH` is blank, and the script hits:

```bash
warn "Could not fetch Go checksum for verification. Proceeding with install."
```

Two fixes:

1. Change the URL to `https://go.dev/dl/?mode=json&include=all` which returns all historical releases (larger response, but the grep still works).
2. Hardcode the SHA256 hash for the specific pinned version directly in the script — simpler, doesn't depend on a second network request, but requires updating when bumping `GO_VERSION`.
