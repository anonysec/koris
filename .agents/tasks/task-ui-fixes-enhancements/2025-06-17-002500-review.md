# UI fixes and admin panel enhancements (v1)

Topbar reworked to remove the realtime status dot and replace the expanding search input with a static command-palette trigger showing a platform-aware shortcut. Sidebar gains a dedicated Tickets nav item. Settings simplified from 7 tabs to 5 (Panel Status and Audit Logs removed), with Panel Settings wired to an API and Export Backup wired to CSV download endpoints. VPN Cores view expanded with protocol-specific advanced options (comp-lzo, DPD, IKE proposals, SSH session limits, etc.). Backend adds WebSocket pub/sub so customer-created tickets broadcast a notification to connected admin clients. i18n keys added for EN/FA/ZH across the new strings.

**Watch for:** Notification bell navigates to the now-deleted `audit-logs` tab (confirmed). ~30 form field labels in VPN Cores remain hardcoded English, breaking multi-language (confirmed). Telegram settings form is uncontrolled ŌĆö no v-model state, no save call (confirmed). Import backup is a UI stub with no backend (confirmed).

## High-level view

Settings dropped two placeholder tabs and made the remaining five functional where backend endpoints already existed. Panel Settings reads/writes `/api/panel-settings`; Export links open CSV endpoints registered in the Go router. The gaps are Telegram (form with no state/handler) and Import (stub toast). Export uses `window.open()` which may fail silently if auth is header-based rather than cookie-based.

The WebSocket notification pathway is structurally complete: backend `broadcastNotification` fans out to subscriber channels, the frontend realtime store accumulates `Notification` objects, and `notificationCount` is derived. The missing piece is delivery to the user ŌĆö the bell icon still navigates to a removed page instead of surfacing the notifications.

VPN Cores gained substantial protocol options for all four protocols, organized into collapsible `<details>` blocks. The i18n gap is significant: every protocol-specific field label (~30 of them) is a hardcoded English string that won't translate when locale changes.

## Notification pipeline without last-mile delivery

The backend pub/sub is structurally sound: `addWSSubscriber` allocates a buffered channel (capacity 16), `broadcastNotification` fans out with a non-blocking send (messages dropped if buffer full), and `removeWSSubscriber` closes the channel after removal. The realtime WebSocket handler subscribes on connect and defers cleanup, selecting on `notifCh` alongside the existing ticker/ping loop. On the frontend, `useRealtimeStore` handles the `notification` message type, prepends to a capped array (50 items), and exposes `notificationCount` and `markAllRead`.

None of this state is surfaced to the admin. `AppShell.handleNotifications` still contains:

```ts
function handleNotifications() {
  // Navigate to settings audit logs which shows system events
  // TODO: Replace with proper notification dropdown component
  router.push({ name: 'settings', params: { tab: 'audit-logs' } })
}
```

The `audit-logs` tab no longer exists. Clicking the bell navigates to Settings which defaults to `panel-settings` since the URL param won't match any tab key. A customer creating a ticket will increment the badge but the admin has no way to see the notification content or mark it read.

## VPN Cores i18n coverage gap

Every protocol-specific field uses a raw string literal:

```html
<KFormField :name="`${proto}-transport`" label="Transport">
<KFormField :name="`${proto}-cipher`" label="Cipher">
<KFormField :name="`${proto}-tls`" label="TLS Mode">
```

The pattern repeats across all collapsible "Protocol Options" and "Advanced Settings" sections (Topology, Keepalive, Fragment, MSS Fix, Verbosity, Comp-LZO, Push Routes, Custom Directives, LCP Echo Interval, LCP Echo Failure, Idle Timeout, Refuse CHAP, Refuse PAP, Require MS-CHAPv2, DPD Interval, DPD Timeout, Rekey Time, IKE Proposals, ESP Proposals, Left ID, Right ID, Fragment Size, Max Sessions, Idle Timeout, Shell Access, Allowed Keys, MTU, Max Clients, Conn Limit, Enable Logs) ŌĆö roughly 30 labels that stay English regardless of locale.

Confirm dialogs (`handleDeleteNode`) and the token banner also use inline English strings outside `t()`.

## Telegram form is decorative

The Telegram Bot tab renders two `KInput` components with no `v-model` binding:

```html
<KInput :id="fieldId" placeholder="123456:ABC-DEF..." type="password" />
<KInput :id="fieldId" placeholder="-1001234567890" />
```

No `ref` captures the values, and the "Save Telegram Settings" button has no `@click` handler. The form collects nothing and submits nothing.

## Export auth assumption

Export Backup calls `window.open(url, '_blank')` for each CSV endpoint. This works only if the browser sends session cookies with the navigation request. If admin authentication relies on `Authorization` headers (as with token-based auth), the opened tab would receive a 401. The Go routes are protected by `requireAdmin` ŌĆö confirm that the auth check accepts cookies, not just headers.
